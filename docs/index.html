THE STACK

What we chose, and why it's there

Vision / Core ML

Object detection + fallback segmentation

Fast on-device detection with YOLO models. Bounding boxes alone were too rough for placing a face, so detection is the first step — never the whole answer.

SAM2.1 Tiny (Core ML)

Tap-to-segment object mask

Apple's SAM2 port gives a clean mask of the exact object a kid taps. It's heavy, so we run it on tap only, prewarm it, cache results, and fall back to Vision if it fails.

ARKit + RealityKit

The face on the real object

The segmentation mask tells AR exactly where the eyes and mouth belong. The mouth animates word-by-word while the object speaks — if it drifts out of sync, the illusion dies.

Apple Speech

Speech recognition (ASR)

Turns a kid's spoken question into text. We run it as a controlled mode — listen, transcribe, stop, send — and mute TTS first so the app never listens to itself.

Foundation model + local VLM

Personality and conversation

A language model gives each object its personality and answers. The on-device model only sees objects at a general level, so a VLM on a Mac over the local network fills in the finer details.

ElevenLabs TTS

The character voice

Apple's built-in voices sound flat, and for kids the voice is the personality. ElevenLabs gives each object an expressive voice, with AVSpeechSynthesizer as the offline fallback.

TRIED & DROPPED

Dead ends that taught us something

DROPPED

Bounding boxes only

Not accurate enough to place a face nicely on a real object. Segmentation gives the actual shape — the object feels genuinely "selected."

DEMOTED

Apple TTS as the main voice

Reliable and Apple-first, but too flat for a character kids should fall in love with. Kept as the fallback — demoted, not fired.
